| 구분                  | GET 방식                                            | POST 방식                       |
| -----------------    | ---------------------------------------             | ----------------------------- |
| 데이터 전달 위치       | URL 쿼리 문자열 (`?key1=value1&key2=value2`)         | HTTP 메시지 본문(body)에 포함         |
| 주된 용도             | 리소스 조회(읽기 전용)                                 | 리소스 생성·수정·삭제(상태 변경)           |
| 안전성 (Safety)       | 서버 상태 변경을 일으키지 않도록 설계되어 있음 (읽기 전용) | 서버 상태를 변경할 목적으로 사용            |
| 멱등성 (Idempotency)  | 동일 요청을 여러 번 보내도 결과가 동일                   | 요청을 반복하면 서버 상태가 여러 번 변경될 수 있음 |
| 캐싱 가능 여부         | 기본적으로 캐시 가능                                   | 기본적으로 캐시되지 않음 (특별 설정 시 가능)    |
| URL 길이 제한 및 보안  | URL 길이 제한(브라우저·서버마다 상이), 민감 정보 노출 위험 | 본문에 담으므로 대용량 및 민감 정보 전송에 더 안전 |
| 대용량 데이터 전송 적합성| 부적합                                               | 적합                            |
| 실무 활용 예시          | 검색, 페이지 네비게이션, 필터링 등                      | 회원가입, 로그인, 글쓰기, 파일 업로드 등      |
